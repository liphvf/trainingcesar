Implementando a interface java.util.Collection

Conteúdo

  • TDD - Test Driven Development
  • JUnit
  • Geração e refatoração de código
  • Cobertura de código

Estudo de caso

Parabéns! Você foi contratado pela Oracle e, como primeira atribuição, foi encarregado de implementar a interface java.util.Collection.

  • Qual método de desenvolvimento você utilizará?
  • Como você garantirá a correção de sua implementação?
    • Lembre-se que milhões de desenvolvedores utilizarão sua implementação!
  • Que tal começar escrevendo um teste?

Roteiro

  1. Crie o teste emptyCollectionSizeTest.
  2. Modifique o código para passar no teste emptyCollectionSizeTest.
  3. Crie os testes singletonCollectionSizeTest, addToEmptyCollectionTest e emptyCollectionIteratorTest.
  4. Modifique o código para passar nos testes singletonCollectionSizeTest, addToEmptyCollectionTest e emptyCollectionIteratorTest.

Leitura

  • Leia o capítulo Test-Driven Development do livro Test-Driven Development: A Practical Guide.
  • Leia a introdução do livro Pragmatic Unit Testing in Java with JUnit.

Exercícios

  1. Implemente um método que, dada uma base de cálculo anual em R$, calcule o imposto sobre a renda da pessoa física para o exercício de 2012, ano-calendário de 2011.
    • Consulte a Tabela Progressiva para Cálculo anual do Imposto sobre a Renda da Pessoa Física - a partir do exercício de 2012.
    • Lembre-se de começar pelos testes!
  2. Conclua a implementação da interface java.util.Collection.
    • Consulte a view Tasks do Eclipse (Window > Show View > Tasks).
    • Lembre-se de começar pelos testes!
  3. Desenvolva uma nova implementação da interface java.util.Collection que instancia um novo vetor de objetos com capacidade para mais dez elementos no lugar de apenas um.
    • Reutilize parte do código da implementação original, se possível.
    • Você precisará desenvolver novos testes? Em caso positivo, lembre-se de começar por eles!
  4. Implemente a interface java.util.Map.
    • Lembre-se de começar pelos testes!
